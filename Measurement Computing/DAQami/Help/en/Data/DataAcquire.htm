Viewing and Logging Data

DAQami can acquire, log, and present data for viewing on displays.

Acquired data can be exported to .csv when an acquisition is stopped.

Data from activated channels can be logged to a Technical Data Management Streaming (TDMS) file. You can specify the name of the data file and the location to save the data from the Logging Options dialog box.

Displays can also export data that is currently showing on the display.

While viewing data on displays, you can click on the lock icon on each display to toggle the display between live and review mode.

When the lock icon is locked, the display is in live mode and updates as data is acquired based on the configuration.

Click on the icon to change to review mode.
When the lock icon is unlocked, the display is in review mode. A horizontal scrollbar appears on the display for reviewing samples. Data continues to be acquired based on the configuration.

Click on the icon to change to live mode.

Use the buttons on the DAQami main window to control when data is acquired and when it is acquired and logged:

Start acquiring and logging data from all active channels to a TDMS file. Optionally, you can view up to 1 million samples per channel on a display, and export data to .csv file.

Note: When continuously logging more than 1 million samples per channel, you may experience performance issues depending on the PC that is running DAQami, the number of channels being sampled, the sample rate, and other factors. For optimal performance in this scenario, reduce the sample rate, reduce the number of active channels, and/or install a faster hard drive.
Start acquiring data and, optionally, view up to 1 million samples per channel on a display, and export data to .csv file. No data is logged to a TDMS file.
Stop acquiring data. Data can be exported to .csv once the acquisition stops.

The timestamp applies to the first channel in the acquisition.
